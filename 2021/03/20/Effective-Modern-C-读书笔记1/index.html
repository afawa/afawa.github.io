在编译时，编译器会使用expr对于两种类型：T和ParamType。这种在模板中进行的类型推断一般分为三种情况：

类型1：ParamType是引用或者指针，但不是Universal Reference

首先假设ParamType是一种引用类型，在这种情况下类型推断按照下面的方式工作：

  1. 如果expr的类型是引用，忽略引用部分
  2. 对于expr的类型和ParamType进行模式匹配(pattern-match)来决定T

如果param是一个指针，推断方式相似

类型2：ParamType是Universal Reference

在这种情况下类型推断按照下面的方式工作：

  1. 如果expr是左值，T和ParamType都被推断为左值引用。
  2. 如果expr是右值，“常规”（类型1）法则适用

类型3：ParamType既不是指针也不是引用

在这种情况下类型推断按照下面的方式工作：

  1. 如果expr的类型是引用，忽略引用部分
  2. 忽略expr的const和volatile等修饰词
